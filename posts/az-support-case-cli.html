Creating an Azure support case without a subscription

10 Sept 2024

I recently lost access to one of my Azure subscriptions and could no longer see it in the subscription list in the Azure Portal or using the Azure CLI.

Of course you can create a support case from the Azure portal but if you don't have a subscription, you need to use the Azure CLI to create it.

Here's how you can create an Azure support case using the Azure CLI:

  1. Install the Azure CLI:
    2. winget install Azure.CLI
  2. Install the support extension:
    3. az extensions add --name support
  3. Get the ID for the Subscription Management service:
    4. az support services list --query "[?contains(displayName, 'Subscription management')]"
  4. Get the ID for the "Unable to access my subscription / Issues signing in or accessing my subscriptions" problem category:
    5. az support services problem-classifications list --service-name [GUID from previous result here]"
  5. Get the "A-3" country code from this page: List of ISO 3166 country codes
  6. Get the "Name of Time Zone" from this page: Microsoft Time Zone Index Values
  7. Create the support case:
    8. az support no-subscription tickets create --ticket-name "CannotAccessSub" --title "I cannot access my subscription" --contact-country "[Your A-3 Country Code]" --contact-email "[Your email address]" --contact-first-name "[Your first name]" --contact-language "en-us" --contact-last-name "[Your last name]" --contact-method "email" --contact-timezone "[Your time zone name]" --description "I can no longer see my Azure subscription listed in the portal or CLI. Sub Name=[Your Sub Name Here] Sub ID=[Your Sub ID Here]" --advanced-diagnostic-consent "Yes" --problem-classification "[The problem classification ID you got earlier]" --severity "minimal"
